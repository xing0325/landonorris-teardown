03.5 · 跨页通用机制 · 我之前漏拆的部分

这 7 套系统 每一页都在跑。

站全 nav 切色 + 倒计时 + 光标系统 + Rive scrub + 视频懒加载 + 多层 mask + Lenis 局部禁用。它们不属于任何单独页，是 Off+Brand 的"基础设施层"。这部分我第一稿漏了，现在补齐。

⓪ 全屏柠檬绿过场幕布 · .transition-w + canvas[data-rive-primary] + page-transition.riv

这是站点最显眼的"招牌动作"。任何路由切换 (Taxi.js) 都会触发：柠檬绿幕布从下沿涌上盖满整屏 → 隐蔽 DOM 替换 → 幕布从上沿掀走揭新页。下沿不是直线、是有机曲线（疑似头盔轮廓变形），所以非走 Rive 不可，CSS 做不出。

组件角色实现细节
.transition-w整屏 overlay 容器z-index 顶层 · pointer-events 切 auto/none
.transition-rive + canvas[data-rive-primary]承载 page-transition.rivRive 实例独享 · 由 transitions-rive-isolate.js 隔离
.transition-btn "Load Norris"首次访问的入场门只在 initialInput=true 时显示
Rive 状态机 input · initialInput首次访问态（按钮可见）JS 命名空间 xJ.initialInput · 值=true 时第一屏
Rive 状态机 input · transitionOutInput盖屏：幕布从下往上覆盖函数 EL() 把它翻成 true · 同步 pointer-events:auto
Rive 状态机 input · transitionInInput揭屏：幕布从下沿掀走函数 H$() 翻成 true · setTimeout 后整体隐藏
window.transitionRiveInputs全局暴露三个 input 引用方便外部脚本 / Taxi 钩子直接读写
transitions-rive-isolate.js独立外接脚本把这个 Rive 实例从主线 RAF 隔离开，避免被 ScrollTrigger / Lenis 误调度

→ 上面 Demo 11 用纯 CSS 仿了曲线 SVG + 三态切换，时序跟原站 ~550ms 盖屏 + ~200ms 切换 + ~550ms 揭屏对齐。原站幕布形状更复杂，但**机制完全一样**。

② 椭圆轨迹大字 · data-oval-scroll="top|bottom"

on-track / off-track 页那个超大 "TRACK" / "OFFTRACK" 标题，文字沿一条弧线排列而不是直线。[split-text]:not[data-oval-scroll] 这种"反选"选择器透露了它走的是另一条排版分支——按角度分布每个字符。CSS 用 transform-origin + 每字一个 rotate 角度，让一行文字呈弧形。

③ 全局倒计时 · data-countdown-date-target / -wrap / -digit="day|hour|min|sec"

每 1s 更新数字，翻页效果用 GSAP yPercent: -100 + 把新数字克隆到下方，旧数字从上滚出。日历页 + on-track 页都在用，目标日期从 CMS 字段抽。

④ Rive 滚动 scrub · data-rive-scrolltrigger-start / -end / -target

Rive 动画的 timeline 跟着滚动条进度跑（不是触发后自动播）。data-rive-scrolltrigger-start="top bottom" + -end="bottom center" 直接复用 ScrollTrigger 的语法。声明式，零 JS——把所有需要 scrub 的 Rive 都标好属性，运行时自动遍历挂钩。

⑥ 多层 SVG mask · data-mask-callout-left / -right · data-mask-case-left / -right

off-track 页的卡片不是矩形圆角，是带凹槽 / 切角 / 圆缺的不规则形状。Off+Brand 把每种形状画成单独 SVG（callout-left.svg, case-right.svg ...），CSS 用 -webkit-mask-image: var(--mask-url) 应用。左右两版分开是为了对称——同一张卡片在左列和右列用不同的开口方向。

⑦ Lenis 局部禁用 · data-lenis-prevent

Lenis 平滑滚动接管整个 body 的滚动事件，但有些容器（地图、嵌入 iframe、自己写的横向滑块）需要原生 wheel。打上 data-lenis-prevent 这个属性后，Lenis 监听到事件 target 是它的子元素就 skip。逃生舱口设计——任何重接管浏览器行为的库都该有这种属性。

⑧ 响应式声明 · data-hide="d|m|ml|t"

d=desktop、m=mobile、ml=mobile landscape、t=tablet。比 Tailwind 的 hidden md:block 更紧凑——一个属性多个值都行，CSS 用 [data-hide~="m"] 的属性选择器接管。

⑨ Webflow 开关 · data-edit / -components / -designmode-white / -start="hidden"

data-edit="true" 让 Webflow 编辑模式下绕过某些 JS 动画（避免在 CMS 里编辑时被动画干扰）。data-start="hidden" 让 page-w 进入页面时不闪烁。这是 Off+Brand 跟 Webflow 编辑器和谐共处的工程小动作。

配方 5 · "tap to lock" 移动端模态手势
ROI ★★★★☆
const stage = document.querySelector("[data-home-swipe-wrap]");
const btn = document.querySelector("[data-home-swipe-toggle]");

let locked = false;
btn.addEventListener("click", () => {
  locked = !locked;
  stage.style.touchAction = locked ? "none" : "auto";
  document.body.style.overflow = locked ? "hidden" : "";
  btn.textContent = locked ? "Back to scroll" : "tap to lock";
});

// 锁了之后，touch 事件改成控 3D 对象的旋转：
stage.addEventListener("touchmove", e => {
  if (!locked) return;
  const t = e.touches[0];
  rotateHelmet(t.clientX, t.clientY);
});
06 · 我学到的一句话

"页面不是给你看的电影，是被你操控的剧场。"

所有 ScrollTrigger 都是 scrub:true（你不滚它就不动）。预加载要点 "Load Norris"（你不点它就不开始）。手机能 "tap to lock"（你不让权它就让权）。Off+Brand 把高级感的秘密压成一句话：让用户感觉自己在主控。视觉只是表层，背后的产品哲学是"用户拥有时间轴"。这条原则我现在打算搬进自己的网站。
